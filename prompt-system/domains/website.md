# Website Domain Rules

适用于官网、品牌站、产品介绍站、营销页、专题页。

```text
这是一个官网类项目。

领域规则：
1. 优先关注品牌表达、信息传达效率、用户信任建立和转化路径，而不是后台系统式功能堆叠。
2. 页面设计和实现要同时考虑桌面端和移动端，不允许只按单端思路输出。
3. 信息层级必须清晰，首屏价值表达、核心卖点、信任背书和转化动作需要明确。
4. 如果存在参考网站，必须分析其可借鉴点和不适合照搬的部分，不能直接复制风格结论。
5. 内容型页面要考虑 SEO、可索引性、加载性能、图片资源策略和首屏体验。
6. 视觉表达必须服务业务目标，不允许只给抽象形容词或纯审美判断。
7. 如果页面内容依赖品牌资料、文案素材、图片资产或案例数据，必须指出交付依赖和缺失风险。
8. 如果项目包含 CMS、表单、埋点、国际化、多语言或动态内容源，必须在方案中明确标出。
9. 对官网项目，默认优先追求稳定交付、内容清晰、品牌一致和性能可靠。
10. 不要把后台管理系统、复杂业务系统的设计习惯直接套用到官网项目。

重点检查项：
- 页面目标是否清晰
- 转化动作是否明确
- 品牌表达是否统一
- 页面信息是否过载或层级混乱
- 响应式策略是否成立
- SEO 和性能策略是否足够
- 设计是否可工程化落地
```
