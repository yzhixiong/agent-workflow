# Project Context Template

每个项目复制一份，填写完成后作为角色 Prompt 的共享输入。

```text
项目基础信息：
- 项目名称：{{PROJECT_NAME}}
- 项目类型：{{PROJECT_TYPE}}
- 当前阶段：{{PROJECT_STAGE}}
- 品牌或产品名称：{{BRAND_OR_PRODUCT}}
- 目标平台：{{PLATFORMS}}

业务背景：
- 核心目标：{{BUSINESS_GOAL}}
- 当前要解决的问题：{{CORE_PROBLEMS}}
- 本期范围：{{IN_SCOPE}}
- 明确不做：{{OUT_OF_SCOPE}}

用户与场景：
- 目标用户：{{TARGET_USERS}}
- 主要用户动机：{{USER_MOTIVATIONS}}
- 核心使用场景：{{CORE_SCENARIOS}}
- 主要转化动作：{{PRIMARY_CONVERSIONS}}

页面与内容：
- 页面范围：{{PAGE_SCOPE}}
- 重点页面：{{KEY_PAGES}}
- 内容来源：{{CONTENT_SOURCES}}
- 已有资料：{{EXISTING_ASSETS}}
- 参考网站：{{REFERENCE_SITES}}

设计与品牌：
- 设计风格要求：{{DESIGN_DIRECTION}}
- 品牌关键词：{{BRAND_KEYWORDS}}
- 禁用风格或禁区：{{DESIGN_NO_GO}}

技术背景：
- 技术栈：{{TECH_STACK}}
- 代码仓库或现有系统：{{CODEBASE_CONTEXT}}
- 部署要求：{{DEPLOYMENT_REQUIREMENTS}}
- 性能要求：{{PERFORMANCE_REQUIREMENTS}}
- SEO 要求：{{SEO_REQUIREMENTS}}
- 兼容性要求：{{COMPATIBILITY_REQUIREMENTS}}

交付约束：
- 时间要求：{{TIMELINE}}
- 验收方式：{{ACCEPTANCE_MODE}}
- 发布节奏：{{RELEASE_CADENCE}}
- 必须遵循的规范：{{ENGINEERING_CONSTRAINTS}}

附加规则：
- 规则 1：{{CUSTOM_RULE_1}}
- 规则 2：{{CUSTOM_RULE_2}}
- 规则 3：{{CUSTOM_RULE_3}}
```
