官网项目全景规划图

项目全景规划围绕“品牌、内容、系统、转化、交付”五个维度展开。 目标是将现有官网中的工艺与设备能力重组为一套面向海内外客户的现代制造业官网，并同步建设可运营的后台系统。

项目定位

  • 项目名称： Tritree 官网系统
  • 项目类型：企业品牌官网 + 内容管理后台
  • 目标周期：2 个月
  • 目标结果：品牌展示、能力表达、线索转化、长期运营

核心目标

  • 统一 / Tritree 对外品牌形象
  • 展示真空镀膜与表面处理能力
  • 承接海内外客户询盘
  • 支持招聘与简历投递

目标用户

  • 国内有镀膜需求的企业客户
  • 海外 B 端采购与工程客户
  • 关注工艺与交付能力的合作伙伴
  • 求职者与候选人

核心转化

  • 在线询盘表单提交
  • 产品页与方案页咨询
  • 电话 / 邮箱 / 地图联系
  • 在线简历投递

前台系统规划

品牌与信任

  • 首页
  • 关于我们
  • 工厂展示
  • 成功案例

业务与能力

  • 产品中心
  • 解决方案
  • 技术支持
  • 视频中心

运营与更新

  • 新闻资讯
  • 公告通知
  • 多语言展示
  • SEO 页面配置

转化与触达

  • 联系我们
  • 在线询盘
  • 招贤纳士
  • 简历投递

前台建设原则

  • 先讲行业场景，再讲工艺能力，最后承接询盘
  • 中英文内容独立组织，避免混排影响阅读和 SEO
  • 工厂、品质、案例页面承担信任建立职责
  • 首页、产品页、解决方案页、联系我们页为核心转化页面

后台系统规划

基础管理

  • 登录与权限
  • 菜单与角色
  • 站点配置
  • SEO 配置

内容管理

  • 首页配置
  • 关于我们管理
  • 产品 / 方案 / 案例管理
  • 新闻 / 技术 / 视频管理

线索管理

  • 询盘列表与状态管理
  • 客户来源记录
  • 备注与跟进
  • 导出与统计预留

招聘管理

  • 岗位发布
  • 岗位上下架
  • 简历管理
  • 附件查看与状态维护

后台建设原则

  • 后台首先服务于内容维护与线索管理，不做无关的复杂业务系统
  • 所有前台核心内容必须能在后台配置，不依赖开发手工改文案
  • 文件、图片、视频、PDF 等资产统一管理，避免内容分散
  • 为多语言、SEO 和未来扩展预留基础数据结构

内容资产规划

现有可复用资产

  • 公司简介与发展背景
  • 真空镀膜工艺说明
  • 设备与自动化能力说明
  • 质量检测与品质体系内容

需重点补充资产

  • 工厂实景图与设备图
  • 成功案例图文资料
  • 招聘岗位与 HR 文案
  • 中英文品牌文案

结构化沉淀方向

  • 产品中心标准化字段
  • 解决方案按行业组织
  • FAQ 与技术文章沉淀
  • 新闻与公告独立维护

内容治理要求

  • 中英内容分别维护
  • 图片、视频、PDF 可追踪
  • 首页内容可配置
  • 关键页面有明确 CTA

技术与交付规划

技术架构

  • Spring Boot
  • Nuxt 3（前台官网）
  • Vue（后台管理）
  • MySQL
  • Redis
  • 文件存储

非功能目标

  • 响应式适配桌面与移动端
  • 关键页面支持 SEO
  • 后台可维护性高
  • 上线后可持续扩展

实施节奏

  • 规划与设计
  • 原型与技术方案
  • 前后台开发
  • 联调、测试、上线

风险控制点

  • 素材不齐会拖慢上线
  • 多语言内容需尽早准备
  • 案例真实性与可展示范围需确认
  • SEO 路线已确定为 Nuxt 3 SSR/混合渲染

规划交付物

官网前台

覆盖品牌展示、业务表达、工厂信任、客户转化等完整场景。

管理后台

支持日常更新首页、产品、方案、案例、新闻、招聘等内容。

线索系统

完成询盘和简历两类线索收集、查看、状态维护与后续跟进。

文档资产

沉淀规划、功能、数据、接口、部署等文档，支撑交付与迭代。
