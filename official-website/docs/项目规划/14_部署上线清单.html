官网部署上线清单

本文档用于规范上线前准备、发布过程、发布后巡检和回滚预案。官网项目虽然规模不大，但上线环节如果无清单，最容易在域名、HTTPS、配置或数据初始化上出错。

上线前准备

检查项 内容 优先级 责任方
服务器准备应用服务器、数据库、Redis、文件存储环境可用P0开发/运维
域名与 DNS生产域名解析完成，www 与主域名策略明确P0运维
HTTPS 证书SSL 证书安装完成，HTTP 自动跳转 HTTPSP0运维
环境变量数据库、Redis、文件存储、接口地址、上传路径等配置正确P0开发
数据库初始化表结构、基础字典、管理员账号、站点基础配置初始化P0后端
内容准备首页、关于、产品、方案、联系方式等核心内容已录入P0运营/业务

发布过程清单

步骤 内容 优先级 备注
备份备份数据库、配置文件、历史部署包P0确保可回滚
部署后端发布 Spring Boot 服务，确认启动成功P0检查日志无异常
部署前台发布前台与后台静态资源，更新 Nginx 配置P0确保缓存策略合理
配置校验检查 API 地址、上传地址、跨域、反向代理配置P0常见故障点
冒烟测试首页访问、后台登录、询盘提交、文件上传、招聘投递P0发布后立即执行

上线后巡检

检查项 内容 优先级 备注
前台访问PC 和移动端主要页面可访问，页面无空白或资源 404P0发布后首轮检查
后台访问登录成功、菜单正常、内容可编辑P0验证运维交付
表单链路询盘和简历可正常提交并入库P0必须人工验证
SEO 基础页面标题、描述、robots、站点地图配置检查P1上线后可再复核
日志与监控查看错误日志、异常告警、服务状态P1至少保留首日监控
上线文档必须和实际部署方式保持同步。每次部署方案有变化时，应同步更新这份清单， 否则清单会很快失效，反而误导发布流程。
