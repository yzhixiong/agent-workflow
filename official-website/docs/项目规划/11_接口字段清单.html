官网接口字段清单

本文档在接口规划基础上，进一步列出高优先级接口的核心字段建议，供前后端联调前对齐。 这里强调的是“首批必须实现”的字段，不追求一次性覆盖所有可选扩展字段。 除非特别说明，下表中的响应字段均指统一返回结构 `data` 内部字段。

前台核心接口字段

接口 请求字段 data字段 备注
GET /api/public/home lang banners、company_intro、highlight_metrics、business_blocks、featured_cases、news_list、contact_block 首页聚合接口
GET /api/public/products lang、category_id、keyword、page_no、page_size records[id,name,cover,summary,category_name]、total 产品列表页
GET /api/public/products/{id} path id、lang id、name、cover、gallery、summary、materials、colors、performances、applications、faq 产品详情页
GET /api/public/solutions/{id} path id、lang id、name、cover、pain_points、advantages、related_products、related_cases、cta 解决方案详情
POST /api/public/inquiries company_name、contact_name、email、phone、country、inquiry_type、content、file_ids[] success、inquiry_id 询盘提交
POST /api/public/resumes job_id、name、phone、email、intro_text、resume_file_id success、resume_id 简历投递

后台核心接口字段

接口 请求字段 data字段 备注
POST /api/admin/auth/login username、password token、user_info[id,username,nickname,roles] 后台登录
GET /api/admin/products keyword、category_id、status、page_no、page_size records、total 后台产品列表
POST /api/admin/products category_id、name_zh、name_en、cover_file_id、summary_zh、summary_en、detail_fields... id 产品新增
POST /api/admin/solutions name_zh、name_en、cover_file_id、pain_points_html、advantages_html、product_ids[]、case_ids[] id 方案新增
GET /api/admin/inquiries status、keyword、page_no、page_size records[id,company_name,contact_name,email,phone,country,status,created_at]、total 询盘管理列表
PATCH /api/admin/inquiries/{id} status、remark success 询盘状态维护
POST /api/admin/files/upload multipart file、biz_type file_id、url、file_name 统一文件上传
首期联调建议先锁定 P0 页面与表单链路相关接口字段，尤其是首页聚合、产品详情、解决方案详情、询盘提交和简历投递。 其他字段可在保证接口兼容的前提下逐步扩展。
