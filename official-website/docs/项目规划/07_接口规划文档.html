官网接口规划文档

接口规划建议分为前台公开接口、后台管理接口、文件接口和认证接口四层。 前台接口以只读为主，承接页面渲染和表单提交；后台接口以 CRUD、排序、上下架、状态管理为主。

接口分层建议

接口层 路径前缀 用途 备注
前台公开接口 /api/public/* 前台页面内容读取、询盘提交、简历投递 无需登录，限流与参数校验必做
后台认证接口 /api/admin/auth/* 登录、退出、获取当前用户、刷新 token JWT 或 Session + Redis
后台管理接口 /api/admin/* 后台内容管理、线索管理、站点管理、权限管理 全部需要鉴权
文件接口 /api/admin/files/* 文件上传、查询、删除、引用 后台上传为主，前台仅访问 URL

前端渲染与语言路径约定

项 约定 说明
官网前台技术Nuxt 3 SSR / 混合渲染前台页面在服务端请求公开接口并输出完整 HTML，兼顾 SEO 与动态内容。
后台管理技术Vue SPA后台以管理效率为主，不承担 SEO 诉求。
语言路径/zh-cn/* / /en/*中英文页面使用独立 URL 路径，便于 SEO、hreflang 和内容隔离。
接口调用方式服务端渲染 + 浏览器交互复用同一组公开接口SSR 首屏和后续前端交互均调用 /api/public/*，后端无需区分静态或动态页面。

认证与响应建议

项 建议 说明
认证方式 后台使用 JWT + Redis 或 Session + Redis 按团队习惯选型，重点是登录态可控与失效管理。
响应格式 统一使用 code、message、data 结构 便于前端全局拦截与错误提示。
分页格式 records、page_no、page_size、total 后台列表统一风格。
语言参数 支持 lang=zh-CN / lang=en-US 前台公开接口建议统一语言入参。
文件上传 先传文件，返回 file_id，再关联业务表 有利于统一资源管理。
安全要求 参数校验、限流、防刷、上传类型校验 询盘和简历接口尤其需要加强控制。
接口规划阶段不建议一次性写满所有请求和响应字段。 更合理的做法是先把资源边界和接口组稳定下来，再在联调前为高优先级模块补详细字段定义。 当前前端技术路线已确认，因此公开接口需要兼顾 Nuxt 3 SSR 首屏渲染和浏览器侧交互复用。
